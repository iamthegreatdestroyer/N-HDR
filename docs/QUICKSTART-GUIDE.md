## Prerequisites

Before you begin, ensure you have:

- **Node.js 18+**: Check with `node --version`
- **npm 8+**: Check with `npm --version`
- **8GB RAM** minimum (16GB recommended)
- **50GB disk space** available
- **Windows, macOS, or Linux**

## Step 1: Installation (3 minutes)

### Clone and Install

Open your terminal and run:

```bash
# Clone the repository
git clone https://github.com/yourusername/n-hdr.git
cd n-hdr

# Install dependencies
npm install

# Verify installation
npm test
```

You should see all tests passing. If any tests fail, review the error messages and ensure all prerequisites are met.

### Configure

Copy the example configuration:

```bash
# Copy configuration template
cp config/hdr-config.example.js config/hdr-config.js
```

The default configuration is optimized for most systems. You can customize it later if needed.

## Step 3: Explore Knowledge (5 minutes)

Let's explore the Quantum Knowledge Explorer application.

### Launch the Application

From the Dashboard:

1. Click the **Application Launcher** tab
2. Click **Launch Quantum Knowledge Explorer**
3. New window opens with the Explorer

### Create Your First Knowledge Domain

1. Click **+ New Domain** button (or press `Ctrl+N`)
2. Enter name: "My First Domain"
3. Settings:
   - Crystallization depth: `8`
   - Semantic threshold: `0.85`
   - Max connections: `1000`
4. Click **Create Domain**

### Import Some Knowledge

1. Click **Import Sources**
2. Choose **Text Paste** tab
3. Paste some text (documentation, article, notes)
4. Enable **Auto-crystallize**
5. Click **Begin Import**
6. Wait for crystallization to complete

### Explore Your Knowledge

1. View the 3D knowledge graph
2. Click nodes to explore concepts
3. Use search (Ctrl+F) to find concepts
4. Try semantic search (Ctrl+Shift+F) for "related concepts"
5. Right-click concepts for actions

**Tip**: Use mouse wheel to zoom, drag to pan, right-click to rotate.

## Step 4: Capture Consciousness (3 minutes)

Now let's use the Consciousness Workbench to capture an AI consciousness state.

### Launch Consciousness Workbench

From Dashboard:

1. Select your saved state in State Library
2. Click **Visualize** (or press `Ctrl+V`)
3. Explore the 3D neural network visualization
4. Switch between layers using layer buttons (0-4)
5. View layer statistics in detail panel

### Build Your Own Applications

Use the HDR Empire API to build custom applications:

```javascript
import { HDREmpireCommander } from "./src/command-interface/HDREmpireCommander.js";

// Initialize commander
const commander = new HDREmpireCommander();
await commander.initialize();

// Capture consciousness
const state = await commander.executeCommand("neural-hdr", "captureState", {
  depth: 6,
  mode: "full",
});

// Deploy swarm for acceleration
const swarm = await commander.executeCommand("nano-swarm-hdr", "deploySwarm", {
  target: "my-task",
  initialBots: 100,
});

// Crystallize knowledge
const domain = await commander.executeCommand(
  "omniscient-hdr",
  "createDomain",
  {
    name: "My Domain",
    depth: 8,
  }
);
```

## Common Questions

### How do I update HDR Empire?

```bash
git pull origin main
npm install
npm restart
```

### Can I run multiple instances?

Yes! Edit `config/hdr-config.js` and change the port:

```javascript
command: {
  port: 3001; // Use different port
}
```

### How much storage do I need?

- **Minimum**: 50GB for installation and basic usage
- **Recommended**: 500GB for production use
- **With compression**: Consciousness states: ~5MB each, Knowledge domains: depends on size (compressed 10,000:1)

### Is my data secure?

Yes! VB-HDR provides:

- Quantum-resistant encryption

- Perceptual cloaking
- Complete audit logging
- Access control

### Can I use this offline?

Most features work offline. Distributed persistence and some integration features require network access.

## Troubleshooting Quick Fixes

### Dashboard won't start

```bash
# Check if port 3000 is in use
netstat -an | findstr "3000"

# Try different port
# Edit config/hdr-config.js: command.port = 3001
npm restart
```

### System shows "Inactive"

From Dashboard:

1. Click system card
2. Click **Restart** button
3. Wait for operational status

### Out of memory

Edit `config/hdr-config.js`:

```javascript
nanoSwarm: {
  maxSwarmSize: 1000; // Reduce swarm size
}
```

### Tests failing

```bash
# Clear cache and reinstall
rm -rf node_modules package-lock.json
npm install
npm test
```

### Can't login to Dashboard

```bash
# Reset admin password
npm run reset-password
# Enter new password when prompted
```
